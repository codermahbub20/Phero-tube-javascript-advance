1.Discuss the scope of var, let, and const ?

Ans: In JavaScript, var, let, and const are used to declare variables, but they have different scopes and behaviors.
var: Function Scope: Variables declared with var are function-scoped. This means they are accessible anywhere within the function in which they are declared, but not outside of it.
Hoisting: Variables declared with var are hoisted to the top of their containing function or global scope. This means you can access a var variable before it's declared in code, but its value will be undefined until the actual declaration.

No Block Scope: Unlike let and const, var does not have block-level scope. This means variables declared with var inside blocks (e.g., if, for, or while statements) are still accessible outside of those blocks.



let: let declarations are block-scoped. This means that a variable declared with let is only visible within the block (enclosed by curly braces) in which it's declared. This includes blocks like if, for, while, and functions.

Variables declared with let are also hoisted, but they are not initialized until the actual declaration statement is encountered. This prevents accessing the variable before it's declared and gives rise to the "Temporal Dead Zone" (TDZ).

let variables cannot be redeclared within the same scope.



const: const is also introduced in ES6, and it is used to declare constants, whose values cannot be reassigned after initialization.

Like let, const variables have block-level scope, but they are also not hoisted.

When you declare a variable with const, you must assign a value to it at the same time, and you cannot reassign it later.

undefined: In JavaScript, null and undefined are two distinct primitive values, and they are often used to represent missing or empty values. While they are similar in some ways, they have different use cases: undefined: undefined is a primitive value in JavaScript that indicates a variable has been declared but has not been assigned a value. It is also the default value of function parameters that are not provided.



NULL: null is a value in JavaScript that represents the intentional absence of any object value or a variable pointing to no object. It is often used to indicate that a variable should be empty or have no value.



3.What do you mean by REST API? ?



REST API, which stands for Representational State Transfer Application Programming Interface, is a set of architectural principles and constraints for designing and interacting with web services. It is a widely used and popular approach for building web services because of its simplicity, scalability, and ease of use. REST is not a specific technology or protocol; rather, it's a set of guidelines and best practices for creating web services that can be easily consumed by clients. Here are the key principles and characteristics of RESTful APIs: Statelessness: Each request from a client to a server must contain all the information needed to understand and process the request. The server should not rely on any previous requests or store session information about the client. This makes REST APIs scalable and easy to cache.

Client-Server Architecture: REST separates the client and server, allowing them to evolve independently. The client and server communicate over a standard interface, typically using HTTP methods.

Uniform Interface: REST APIs have a consistent and uniform interface, which simplifies both client and server components. This uniformity includes the use of standard HTTP methods (GET, POST, PUT, DELETE) for actions and the use of URLs (Uniform Resource Locators) to identify resources.

Resource-Based: Resources are the core concept in REST. Each resource is identified by a unique URI (Uniform Resource Identifier), and clients interact with these resources using standard HTTP methods. Resources can represent data objects, services, or anything else that can be named.

Representation: Resources can have multiple representations, such as JSON, XML, HTML, or plain text. Clients can request the representation that best suits their needs, and servers can send different representations of the same resource based on client requests.

Stateless Communication: REST APIs use stateless communication, meaning each request from the client to the server must contain all the necessary information. Sessions and state information are typically managed on the client side.

Layered System: The architecture can include multiple layers, such as load balancers, caching, and gateways, to improve scalability, security, and performance without affecting the overall system behavior.

Cacheable: Responses from the server can be explicitly marked as cacheable or non-cacheable, which helps improve performance by reducing redundant requests.

Self-Descriptive Messages: Responses should include metadata and hyperlinks to help clients discover and navigate the API. This allows clients to understand the API's capabilities without prior knowledge.

RESTful APIs are commonly used in web and mobile applications to retrieve, create, update, and delete data from servers. They are known for their simplicity and ease of use, making them a popular choice for building modern web services.
